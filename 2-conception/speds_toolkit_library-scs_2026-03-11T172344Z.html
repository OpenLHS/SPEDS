1. Rôle du module

Le module Bibliothèque de fonctions de service de SPEDS est responsable d’offrir des services qui peuvent être nécessaire à l’ensemble des couches du protocole de SPEDS.

2. Implémentation du module

Le module Bibliothèque de fonctions de service de SPEDS est implementé par le produit speds-toolkit-library.

3. Modèle propre au module

[DE-GE-01]

Le module définit la liste de catégorie des algorithmes qui suit :

  1. Algorithme de hachage;

  2. Algorithme de chiffrement symétrique;

  3. Algorithme de chiffrement asymétrique;

  4. Algorithme d’échange de clé de Diffie-Hellman;

  5. Algorithme de signature.

4. Définitions

Bibliothèque de projet 

Bibliothèque permettant de récupérer des informations externes reliées à un projet, notamment les entités et leur clé publique associée.

Bibliothèque de sécurité 

Bibliothèque offrant des services de sécurité respectant les recommandations actuelles. Elle permet de remplir des objectifs de sécurité assurés par des moyens cryptographiques, tels la confidentialité, l’intégrité et l’authentification.

5. Type au module

5.1. Type simple de donnée

Table 4. Types simples généraux
Nom Format Description

SpedsLayer

Énumération

Identifie les couches de SPEDS selon le modèle OSI : APPLICATION, PRESENTATION, SESSION, TRANSPORT, NETWORK et LINK.

AlgorithmCategory

Énumération

Identifie les catégories d’algorithme disponibles au sein de la bibliothèque : HASH, SYMM, ASYM, DH et SIGN.

SecurityProfile

Énumération

Identifie les profils de sécurité pouvant figurer dans le fichier de configuration : STRONG et EFFICIENT

5.2. Type complexe de donnée

S.O.

6. Hypothèse du module

[HYP-001]

Le fichier de configuration contenant les primitives cryptographiques est bien formé et les primitives contenues sont valides.

7. Contrainte du module

[CON-001]

Le module Bibliothèque de fonctions de service de SPEDS est fourni dans le langage de programmation Java.
[CON-002]

Le module Bibliothèque de fonctions de service de SPEDS doit être une façade à la Bibliothèque de projet mais ne doit pas en dépendre directement. Toute interaction doit passer par une interface abstraite fournie à l’initialisation du module. La couche utilisatrice de SPEDS sera responsable d’en fournir une implémentation.
[CON-003]

Le module Bibliothèque de fonctions de service de SPEDS doit être une façade à la Bibliothèque de sécurité. Cette dernière doit être une dépendance au module lors de son initialisation.
[CON-004]

La liste des choix des primitives cryptographiques pour chaque catégorie et chaque couche doit être paramétrable par le biais d’un fichier de configuration. Les catégories sont celles définies dans [DE-GE-01].

8. Fonction du module

Général
[PRO-GE-01] Récupérer le IRI d’une entité

Récupère le IRI d’une entité inscrite dans le projet.
[PRO-GE-02] Vérifier la légitimité d’une entité

Vérifie la légitimité d’une entité définie par un couple (code, puK), où code est le code identifiant l’entité et puK est une clé de chiffrement asymétrique (considérée comme la clé publique de l’entité à légitimer).
[PRO-GE-03] Récupérer la clé publique d’une entité

Récupère la clé publique utilisée par une entité du projet définie par son code identifiant.
[PRO-GE-04] Vérification de l’activité d’un projet

Vérifie si un projet est actif.
[PRO-GE-05] Vérification de l’activité d’un plan de projet

Vérifie si le plan d’un projet est en cours.
Sécurité
[PRO-SE-01] Vérifier la validité d’un certificat

vérifie la validité d’un certificat pour une entité proprement identifiée par une IRI.
Cryptographie
[PRO-CY-01] Effectuer un chiffrement symétrique

Effectue un chiffrement symétrique des données en fonction de la couche qui le demande.
[PRO-CY-02] Effectuer un déchiffrement symétrique

Effectue un déchiffrement symétrique des données en fonction de la couche qui le demande.
[PRO-CY-03] Générer une clé de chiffrement symétrique

Génère une clé pour le chiffrement symétrique des données en fonction de la couche qui le demande.
[PRO-CY-04] Effectuer un chiffrement asymétrique

Effectue un chiffrement asymétrique des données en fonction de la couche qui le demande.
[PRO-CY-05] Effectuer un déchiffrement asymétrique

Effectue un déchiffrement asymétrique des données en fonction de la couche qui le demande.
[PRO-CY-06] Effectuer un hachage

Effectue un hachage des données en fonction de la couche qui le demande.
[PRO-CY-07] Apposer une signature

Appose une signature sur les données en fonction de la couche qui le demande.
[PRO-CY-08] Valider une signature

Valide une signature sur les données en fonction de la couche qui le demande.
[PRO-CY-09] Choisir une valeur pour la méthode d’échange Diffie-Hellman

Choisit une valeur dans un algorithme de poignée de main suivant la méthode Diffie-Hellman en fonction de la couche qui le demande.
[PRO-CY-10] Déterminer la clé secrète pour la méthode d’échange Diffie-Hellman

Détermine la clé secrète dans un algorithme de poignée de main suivant la méthode Diffie-Hellman en fonction des choix effectué par les deux entités en communication et de la couche qui le demande.
[PRO-CY-11] Récupérer l’algorithme disponible pour une catégorie

Indique le nom de l’algorithme disponible (sous une version standardisée) pour une catégorie définie en [DE-GE-01].
[PRO-CY-12] Récupérer les paramètres d’algorithme disponible pour une catégorie

Indique les paramètres par défaut de l’algorithme disponible pour une catégorie définie en [DE-GE-01].
