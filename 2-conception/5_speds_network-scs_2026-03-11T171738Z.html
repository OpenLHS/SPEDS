1. Rôle du module

Le rôle de SPEDS - Réseau, la couche réseau, est d’acheminer les données d’un hôte source à un hôte destination sous un réseau. La topologie du réseau est composée d’un ou plusieurs nœuds (services mandataires) entre la source et destination.

2. Implémentation du module

Le module SPEDS - Réseau est implémenté par le produit speds-network.

3. Modèle propre au module

Le modèle doit assurer un mécanisme qui permet de traiter les signaux de confirmation (primitive de message confirm) et de réponse (primitive de message response) afin de déclencher les signaux associés.

Ceci peut être implémenté par n’importe quelle technologie y compris par un mécanisme de fonctions et de retour de fonctions.

Table 4. Type simple de donnée

Chaîne de caractères représentant un IRI et respectant le format de la norme RFC 3987

Tableau d’octets représentant un sceau résultant de la fonction de signature choisie par la Politique de chiffrement par défaut pour le GRIIS

4.2. Type complexe de données

4.2.1. Unité de donnée du protocole (PDU)

Un type décrivant un format de PDU qui correspond au message transmis entre les couches Réseau des entités. Ce type doit être équivalent au schéma JSON du même objet (cf. annexe Protocol Data Unit de la couche réseau (5)).

4.2.2. Unité de données d’interface (IDU)

Unité d’information transférée en une seule interaction à l’interface de deux couches, constituée d’information de contrôle d’interface (ICI) et de l’unité de données du protocole (PDU).

La IDU avec la couche supérieure est décrite dans l’annexe Interface Data Unit entre la couche transport (4) et la couche réseau (5). La IDU entre la couche NET et la IDU avec la couche inférieure est décrite dans l’annexe Interface Data Unit entre la couche réseau (5) et la couche liaison (6).

5. Hypothèse du module

[HYP-001]

Chaque entité (hôte ou service mandataire) possède un certificat valide et contient une clé privée associée à la clé publique dudit certificat.
[HYP-002]

Un mécanisme qui permet de déterminer l’adresse du prochain nœud existe.
[HYP-003]

La version et la référence du protocole sont connues à l’initialisation du module.

6. Contrainte du module

[CON-001]

Le module doit utiliser la bibliothèque de fonctions de SPEDS (implémentée par speds-toolkit-library) qui contient les primitives de cryptographie utilisée.
[CON-002]

La suite cryptographique (spécifiquement la fonction de hachage, le format des certificats et l’algorithme de signature utilisé) est celle définie par la Politique de chiffrement par défaut pour le GRIIS.

7.1. [PRO-1] Processus « Demande de transfert d’un message »

7.1.1. Interface

Déclencheur Condition

Message reçu de la couche supérieure

  • Le service est transfer

  • La primitive de service est request
flux entrant

  • une requête de transfert de la couche supérieure (IDU)

flux sortant

  • une requête de transfert à la couche inférieure (IDU)

  • une confirmation à la couche supérieure (IDU)

7.1.2. Comportement

[PRO-1.1]

Détermination de l’adresse du prochain nœud

  • Le prochain nœud a pour adresse le IRI définit dans la ICU sous la propriété destination_iri.
[PRO-1.2]

Création d’une PDU pour l’entité distante
[PRO-1.3]

Transmission de la IDU à la couche inférieure.

  1. La ICI est construite avec

    • le IRI de la prochaine destination déterminée en [PRO-1.1]

    • transfert comme service

    • request comme primitive de service

  2. La PDU est construite avec

    • la SDU reçue (propriété message de la IDU) pour contenu ;

    • l’entête suivante :

      • Le type du message est RES.MSG.ENV ;

      • L’identifiant est un UUID déterminé par la couche ;

      • Le IRI de la source originale est celui indiqué par la IDU reçue ;

      • Le IRI du destinataire final est celui indiqué par la IDU reçue ;

      • L’authentification est le certificat disponible encodé au format base64 ;

      • La version doit être celle de la couche Réseau ainsi qu’un IRI vers sa référence ;

      • Les paramètres sont défini par false.

    • les sceaux suivants :

      • Le sceau de le l’entête (header_seal) est la signature numérique de l’empreinte calculée de l’entête du PDU.

      • Le sceau du contenu (content_seal) est la signature numérique de l’empreinte calculée du contenu du PDU.
[PRO-1.3]

Attente de la réception de la confirmation correspondant à cet IDU.
[PRO-1.4]

Transmission de la confirmation avec la même information (propriété message) à la couche supérieure.

7.3. [PRO-3] Processus « Indication de réception d’un message »

7.3.1. Interface

Déclencheur Condition

Message reçu de la couche inférieure

  • Le service est transfer

  • La primitive de service est indication
flux entrant

  • une indication d’un message reçu par la couche inférieure (IDU)

flux sortant

  • une indication d’un message reçu à la couche supérieure (IDU)

  • une réponse vers la couche inférieure (IDU)

7.3.2. Comportement

[PRO-3.1]

Vérifier la signature numérique du IDU

  • Si le sceau d’entête n’est pas une signature numérique valide alors le message est invalide et la réponse est FAILED: Invalid Header Seal.

  • Si le sceau de contenu n’est pas une signature numérique valide alors le message est invalide et la réponse est FAILED: Invalid Content Seal.
[PRO-3.2]

Transmettre le IDU à la couche supérieure s’il est valide

  • Si le message est valide

    • Transmettre le content via le service transfer et la primitive de service indication à la couche supérieure ;

    • Attendre la réception de la réponse correspondant à cet IDU, et conserver la réponse.
[PRO-3.3]

Transmettre la réponse à la couche inférieure

  • Transmettre la réponse obtenue via le service transfer et la primitive de service response

9. Annexes

9.1. Glossaire

Couche inférieure

La couche de communication qui succède à la couche réseau.
Couche supérieure

La couche de communication qui précède la couche réseau.
IDU

Interface Data Unit - Unité d’interface de données. Message pour la communication entre les couches. Est composé d’un ICI (propriété context du JSON) et d’un PDU (propriété message du JSON).
SDU

Service Data Unit - Unité de données du service. Une SDU est l’unité de données transmise par la couche supérieure pour être encapsulée et transmise à une entité distante. Pour une IDU correspondant à une requête de transfer, la SDU est donc la propriété message du IDU transmis par la couche supérieure dans le but d’être traitée.
ICI

Information Control Interface - Information de contrôle de l’interface. Correspond à la propriété context du JSON de la IDU.
PDU

Protocol Data Unit - Unité de données de protocole. La PDU est l’unité de données créée par une couche du modèle OSI pour communiquer avec la couche équivalente sur un autre système. Correspond à la propriété message du JSON de la IDU transmise à la couche inférieure.
PCI

Protocol Control Information - Information de contrôle du protocole. Il s’agit de l’ensemble des informations partagées entre les couches ; l’entête et le pied du message. Correspond aux propriétés header et stamp du PDU de la couche Réseau.
IRI

International resources identifiers - Identifiant de ressource internationalisé balisé par RFC-3987
