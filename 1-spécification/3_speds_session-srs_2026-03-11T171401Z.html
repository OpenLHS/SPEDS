Responsabilité

  1. La couche Session doit s’occuper de l’établissement, la gestion et la terminaison des sessions.

  2. La couche Session doit garantir l’authentification des participants.

  3. La couche Session doit pouvoir reprendre une session en d’interruption (p.ex. panne et restauration de service etc.).

  4. La couche Session doit gérer les cas où le transport (via la couche Transport) échoue.

Les deux derniers points ne sont pas encore assuré.

  1. Les éléments de reprise sont disponibles (état d’une session), mais les points de synchronisation ne sont pas encore défini.

  2. Présentement, les erreurs de la couche de Transport provoquent l’interruption de la communication et la fin de toute communication pour un PGA particulier.

Interfaces

Couche supérieure

entrant

Un message en provenance de la couche Présentation avec un contexte indiquant le code de la source, le code du destinataire, l’identifiant du PGA, la primitive de service et la clé de chiffrement utilisée (SDEK). Le message doit respecter le format standard (décrit après) sinon le message est ignoré.

Services et primitives de service utilisés

  • delegate.request : primitive utilisée par la couche supérieure pour envoyer un message à la couche Session et demander qu’il soit envoyé à son destinataire. Attends un delegate.confirm en réponse.

  • transfer.response : primitive utilisée par la couche supérieure pour indiquer à la couche Session que le message obtenu par transfer.indication est traité. Déclenche un transfer.response à la couche inférieure.

sortant

Un message pour la couche Présentation accompagné d’un contexte indiquant le code de la source, le code du destinataire, l’identifiant du PGA, la primitive de service et la clé de chiffrement utilisé (SDEK). Le message doit respecter le format standard (décrit après).

Services et primitives de service utilisés

  • delegate.confirm : primitive utilisée par la couche courante pour indiquer que le message transmis par delegate.request est pris en charge (avant même son envoi). Est déclenché par un delegate.confirm de la couche inférieure ou par une erreur local. Dans ce dernier cas, le contenu doit alors être de la forme "FAILED: information").

  • transfer.indication : primitive utilisée par la couche Session pour indiquer qu’un message est reçu et doit être transféré. Contient le message. Est déclenché par un transfert.indication de la couche inférieure.

format du contexte d’une communication avec la couche supérieure
{ "SDEK": string
, "source_code": string
, "destination_code": string
, "PGA": string
, "service_primitive": string
, "options": object (actuellement : false)
}

Couche inférieure

entrant

Un message en provenance de la couche Transport avec un contexte indiquant, la primitive de service, le code de la source, le code du destinataire, l’adresse (IRI) de la source et l’adresse (IRI) du destinataire.

Services et primitives de service utilisés

  • transfer.confirm : primitive utilisée par la couche inférieure pour indiquer que le message transmis par transfer.request est transmis à la couche Session du destinataire.

  • transfer.indication : primitive utilisée par la couche inférieure pour indiquer qu’un message est reçu et doit être transféré. Contient le message.

sortant

Un message pour la couche Transport (chaîne de caractère décrivant un objet JSON) avec son contexte indiquant, la primitive de service, le code de la source, le code du destinataire.

Services et primitives de service utilisés

  • transfer.request : primitive utilisée par la couche Session pour envoyer un message à la couche inférieure pour que son transport soit effectué jusqu’à la Session distante. Est déclenché dans le cours de la gestion de session lui même déclenché par un delegate.request de la couche supérieure ou pour transmettre une confirmation dans le cours d’une session.

  • transfer.response : primitive utilisée par la couche Session pour indiquer à la couche inférieure que le message obtenu par transfer.indication est pris en charge. Peut être déclenché après la validation de l’authenticité de la source, auquel cas le contenu est "SUCCEED", ou par une erreur locale. Dans ce dernier cas, le contenu doit alors être de la forme "FAILED: information"). Par exemple, un échec de la validation l’authenticité pourrait déclencher un contenu "FAILED: cannot authentificate the source.".

format du contexte d’une communication avec la couche inférieure
{ "source_code": string
, "destination_code": string
, "source_iri": string
, "destination_iri": string
, "service_primitive": string
, "options": object (actuellement : false)
}

Format des messages pour la couche session

Un message est une chaîne de caractère représentant un objet JSON. Le format de l’objet JSON est défini comme suit.

format d’un message (temporaire – en attente d’un schema)
{ "header" : object
, "stamp" : string
, "content" : string
}
format d’une entête (temporaire – en attente d’un schema)
{ "msgtype" : string
, "id" : string
, "version" : { "number" : "x.x.x"
              , "reference" : "IRI"
              }
, "parameters" : object (actuellement : false)
}

Types

La chaîne de caractères représentant le type du message est l’une des suivantes.

  • SES.PUB.ENV

  • SES.PUB.REC

  • SES.CLE.ENV

  • SES.CLE.REC

  • SES.SAK.ENV

  • SES.SAK.REC

  • SES.MSG.ENV

  • SES.MSG.REC

  • SES.FIN.ENV

  • SES.FIN.REC

Numéro d’identification du message (id)

Le numéro d’identification du message est un numéro généré unique (uuid).

Schéma JSON des communications

  1. Le schéma JSON de la IDU de la couche Présentation (PIDU) ;

  2. Le schéma JSON de la IDU de la couche Session (SIDU) ;

  3. Le schéma JSON de la PDU de la couche Session(SPDU).
