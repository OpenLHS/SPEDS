Spécification de la couche Présentation

Équipe du GRIIS
info@griis.ca
version v7.1.1, 2026-03-11

Responsabilité

  1. La couche Présentation doit gérer le chiffrement et le déchiffrement des messages avec un algorithme de chiffrement symétrique.

  2. La couche Présentation doit créer une clé de chiffrement symétrique (SDEK) si elle reçoit un message par la primitive request.

Interfaces

Couche supérieure

entrant

Un contenu (chaîne de caractère) en provenance d’une couche Application et un contexte indiquant le code de la source, le code du destinataire, le service, la primitive de service et l’identifiant du PGA.

Services et primitives de service utilisés

  • delegate.request : primitive utilisée par la couche supérieure pour envoyer un message à la couche Présentation et demander qu’il soit traité (chiffrement) et que son envoi soit délégué à la couche Session. Attends un delegate.confirm en réponse.

  • transfer.response : primitive utilisée par la couche supérieure pour indiquer à la couche Présentation que le message obtenu par transfer.indication est traité. Déclenche un transfer.response à la couche inférieure.

sortant

Un message de la couche Application (chaîne de caractère décrivant un objet JSON) et son contexte indiquant le code de la source, le code du destinataire, le service, la primitive de service et l’identifiant du PGA.

Services et primitives de service utilisés

  • delegate.confirm : primitive utilisée par la couche courante pour indiquer que le message transmis par delegate.request est traité. Est déclenché par un delegate.confirm de la couche inférieure ou par une erreur local. Dans ce dernier cas, le contenu doit alors être de la forme "FAILED: information").

  • transfer.indication : primitive utilisée par la couche Présentation pour indiquer qu’un message est reçu et doit être transféré. Contient le message. Est déclenché par un transfert.indication de la couche inférieure.

format du contexte d’une communication avec la couche supérieure
{ "source_code": string
, "destination_code": string
, "PGA": string
, "service": string
, "service_primitive": string
, "options": object (actuellement : false)
}

Couche inférieure

entrant

Un message en provenance de la couche Session avec un contexte indiquant le code de la source, le code du destinataire, l’identifiant du PGA, le service, la primitive de service et la clé de chiffrement utilisée (SDEK). Le message doit respecter le format standard (décrit après) sinon le message est ignoré.

Services et primitives de service utilisés

  • delegate.confirm : primitive utilisée par la couche inférieure pour indiquer que le message transmis par delegate.request est traité. Déclenche un delegate.confirm à la couche supérieure.

  • transfer.indication : primitive utilisée par la couche inférieure pour indiquer qu’un message est reçu et doit être transféré. Contient le message.

sortant

Un message pour la couche Session accompagné d’un contexte indiquant le code de la source, le code du destinataire, l’identifiant du PGA, le service, la primitive de service et la clé de chiffrement utilisé (SDEK). Le message doit respecter le format standard (décrit après).

Services et primitives de service utilisés

  • delegate.request : primitive utilisée par la couche Présentation pour envoyer un message à la couche inférieure pour que son envoi soit délégué à la couche Session. Est déclenché par un delegate.request de la couche supérieure.

  • transfer.response : primitive utilisée par la couche Présentation pour indiquer à la couche inférieure que le message obtenu par transfer.indication est traité. Peut être déclenché par une réponse de la couche supérieure (transfer.response), auquel cas le contenu est le même que celui transmis par la couche supérieure, ou par une erreur locale. Dans ce dernier cas, le contenu doit alors être de la forme "FAILED: information"). Par exemple, un problème de déchiffrement pourrait déclencher un "FAILED: decypher failed.".

format du contexte d’une communication avec la couche inférieure
{ "SDEK": string
, "source_code": string
, "destination_code": string
, "PGA": string
, "service": string
, "service_primitive": string
, "options": object (actuellement : false)
}

Format des messages pour la couche présentation

Un message est une chaîne de caractère représentant un objet JSON. Le format de l’objet JSON est défini comme suit.

format d’un message
{ "header" : object
, "content" : string
}
format d’une entête
{ "msgtype" : string
, "id" : string
, "version" : { "number" : "x.x.x"
              , "reference" : "IRI"
              }
, "parameters" : object (actuellement : false)
}

Types

La chaîne de caractères représentant le type du message est

  • PRE.MSG.ENV

Numéro d’identification du message (id)

Le numéro d’identification du message est un numéro généré unique (uuid).

Schéma JSON des communications

  1. Le schéma JSON de la IDU de la couche Application (AIDU) ;

  2. Le schéma JSON de la IDU de la couche Présentation (PIDU) ;

  3. Le schéma JSON de la PDU de la couche Présentation (PPDU).
