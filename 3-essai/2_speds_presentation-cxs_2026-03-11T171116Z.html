2. Contexte général des tests

Le module SPEDS - présentation est responsable de la gestion des messages de la couche présentation. Il doit être capable de recevoir des messages des couches inférieure et supérieure. Les messages reçus doivent être valides et respecter le format attendu. Les messages envoyés doivent être conformes au format attendu.

3. Éléments Testées

[PRO-1] Demande d’échanger des données

Initier une requête pour échanger des données par la couche Présentation.
[PRO-2] Confirme l’échange de données

Confirmer la prise en charge de la requête d’échange de données par la couche Session
[PRO-3] Indique un échange de données

Reçoit la demande d’une requête avec les données échangées par l’initiateur de requête.
[PRO-4] Répond à un échange de données

Répond à un initiateur de requête sous une réponse avec des données à échanger.

6. Cas de Test

6.1. Cas de Test 1 - Envoi réussi d’un envoi de données

Description

Réception d’un message de la couche supérieure par une entité et production d’un message pour la couche inférieure.

Hypothèse

L’entité est démarré.

Données d’entrée
Identifiant Format Description

E_01

InterfaceDataUnit-1_2

Un IDU valide. PrimitiveService est REQUEST. Service est DELEGATE
Données de sortie
Identifiant Format Description

S_01

InterfaceDataUnit-2_3

Un IDU valide.
Test

  1. Appel de la fonction [PRO-1] Demande d’échanger des données

Conditions de réussite

  1. Aucune exception n’est levée.

  2. Il est possible de déchiffrer le content dans le message de InterfaceDataUnit-2_3 (S_01) à l’aide de la clef SDEK.

  3. Le message de sortie est conforme au format attendu.

  4. La PrimitiveService est REQUEST.
