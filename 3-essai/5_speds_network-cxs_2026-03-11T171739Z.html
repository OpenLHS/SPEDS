8.2. [CT_02] Demande de transfert d’un message avec confirmation

Description

Une demande de transfert avec une confirmation associée.

Hypothèse

S.O.

Données d’entrée

  • Entrée 1 - InterfaceDataUnit-4_5 où

    • le service est transfer;

    • la primitive de service est request;

    • l’IRI source est l’IRI de l’hôte source;

    • l’IRI destination est l’IRI de l’hôte de destination;

    • le message est une chaîne de caractères quelconque.

  • Entrée 2 - InterfaceDataUnit-5_6 où

    • le service est transfer;

    • la primitive de service est confirm;

    • l’IRI source est l’IRI de l’hôte source;

    • l’IRI destination est l’IRI de l’hôte de destination;

    • le message est un message réseau en format JSON où

      • l’entête

        • le type de message est RES.MSG.ENV;

        • l’identifiant du message est un UUID;

        • l’IRI source est l’IRI de l’hôte source;

        • l’IRI destination est l’IRI de l’hôte destination;

        • le champs authentification du message est un certificat valide pour l’entité source encodée en Base64;

        • la version correspond à la version v7.1.1 ;

      • sceaux

        • le sceau d’entête est une signature valide de l’entête du message encodée en Base64;

        • le sceau de contenu est une signature valide de l’entête du message encodée en Base64;

      • le contenu est une une chaîne de caractères quelconque.

Données de sortie

  • Sortie 1 - InterfaceDataUnit-5_6 où

    • le service est transfer;

    • la primitive de service est request;

    • l’IRI destination est l’IRI de l’hôte de destination;

    • le message est un message réseau en format JSON où

      • l’entête est identique à l’entête du message réseau de l’entrée 2;

      • les sceaux sont identiques aux sceaux du message réseau de l’entrée 2;

      • le contenu est identique au message de l’entrée 1.

  • Sortie 2 - InterfaceDataUnit-4_5 où

    • le service est transfer;

    • la primitive de service est confirm;

    • l’IRI source est l’IRI de l’hôte source;

    • l’IRI destination est l’IRI de l’hôte de destination;

    • le message est identique au contenu du message réseau de l’entrée 2.

Instruction

  • Exécution de PRO-1 - Demande de transfert d’un message avec l’entrée 1 :

    • Réception de l’entrée 1 par la couche supérieure et transmission d’une InterfaceDataUnit-5_6 à la couche inférieure.

    • Exécution de PRO-2 - Confirmation d’un transfert avec l’entrée 2 :

      • Réception de l’entrée 2 par la couche inférieure et envoi du signal de confirmation.

    • Réception du signal de la confirmation et échange de InterfaceDataUnit-4_5 à la couche supérieure.

Condition de réussite

  • Aucune erreur est soulevée.

  • PRO-1 - Demande de transfert d’un message a tranmis dans un délai maximal de 2 secondes l' InterfaceDataUnit-5_6 et celle-ci correspond à la sortie 1.

  • PRO-1 - Demande de transfert d’un message a échangée dans un délai maximal de 2 secondes l' InterfaceDataUnit-4_5 et celle-ci correspond à la sortie 2.

8.3. [CT_03] Indication de réception d’un message avec un sceau d’entête invalide

Description

Une indication de message avec un sceau d’entête invalide est rejetée.

Hypothèse

S.O.

Données d’entrée

  • Entrée 1 - InterfaceDataUnit-5_6 où

    • le service est transfer;

    • la primitive de service est indication;

    • le message est un message réseau en format JSON où

      • le sceau d’entête du message n’est pas une signature numérique valide avec la clé publique du certificat associée et l’empreinte numérique de l’entête du message.

Données de sortie

  • Sortie 1 - InterfaceDataUnit-5_6 où

    • le service est transfer;

    • la primitive de service est response;

    • l’IRI de destination est l’IRI de l’entité source;

    • le message est FAILED: Invalid Header Seal.

Instruction

  • Exécution de [PRO-3] - Indication de réception d’un message avec l’entrée 1.

Condition de réussite

  • Aucune erreur est soulevée.

  • PRO-3 - Indication de réception d’un message n’a pas échangé une InterfaceDataUnit-4_5 à la couche supérieure.

  • PRO-3 - Indication de réception d’un message a tranmis dans un délai maximal de 2 secondes l' InterfaceDataUnit-5_6 et celle-ci correspond à la sortie 1.
