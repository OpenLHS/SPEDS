3. Contexte général des tests

Ce document définit les essais nécessaires pour vérifier et valider les échanges des interfaces de la couche transport.

4. Éléments Testées

[PRO-04] Demande d’échanger des données

Demande d’échanger des données sous une requête.

[PRO-05] Confirme l’échange de données

Confirme l’échange de données sous une réponse.

[PRO-11] Traite l’échange de données

Traite la demande d’échange de données et valide celle-ci. Quand celle-ci est valide, le receveur de données raporte (indique) les données, et répond une confirmation de la réception des données à l’envoyeur.

5. Hypothèse

[HYP-01]

Il n’y a aucune entrave à la communication pour les messages transmis sur la couche transport.
[HYP-02]

À moins d’indication contraire, les chaînes de caractères reçues en entrées sont réputées être valides et correspondre au schéma JSON attendu.

6. Données globales

Initialisation d’une entité

Nécessite:

  • Les valeurs pour SPEDS fournit par l’environnement où une version de SPEDS et une référence de SPEDS.

Version SPEDS

Une même valeur paramétrable pour tous les cas de tests.

Réference SPEDS

Une même valeur paramétrable pour tous les cas de tests.

8.2.4. [CT_GEN_01] Envoi et réception de messages entre deux entités

Description

Deux entités s’envoient des messages requêtes et réponses.

Hypothèse

  1. Deux entités (une entité Alpha et une entité Beta) sont existantes et peuvent se communiquer.

Données d’entrée
Identifiant Valeur

in-alpha

 
InterfaceDataUnit-3-4:
  "context":
   "source_code":       est le code de l'entité alpha
   "destination_code":  est le code de l'entité beta
   "source_iri":        est l'iri de l'entité alpha
   "destination_iri":   est l'iri de l'entité beta
   "tracking_number":   [non-validé]
   "options":           [non-validé]
  "message":            `Alpha request Beta`
Identifiant Valeur

in-beta

 
InterfaceDataUnit-3-4:
  "context":
   "source_code":       est le code de l'entité beta
   "destination_code":  est le code de l'entité alpha
   "source_iri":        est l'iri de l'entité beta
   "destination_iri":   est l'iri de l'entité alpha
   "tracking_number":   [non-validé]
   "options":           [non-validé]
  "message":            `Beta request alpha`
Données de sortie
Identifiant Valeur

out-alpha

 
InterfaceDataUnit-3-4:
  "context":
   "source_code":       est le code de l'entité beta
   "destination_code":  est le code de l'entité alpha
   "source_iri":        est l'iri de l'entité beta
   "destination_iri":   est l'iri de l'entité alpha
   "tracking_number":   [non-validé]
   "options":           [non-validé]
  "message":            `Beta request alpha`
Identifiant Valeur

out-beta

 
InterfaceDataUnit-3-4:
  "context":
   "source_code":       est le code de l'entité alpha
   "destination_code":  est le code de l'entité beta
   "source_iri":        est l'iri de l'entité alpha
   "destination_iri":   est l'iri de l'entité beta
   "tracking_number":   [non-validé]
   "options":           [non-validé]
  "message":            `Alpha request Beta`
Instructions

  1. Délègue 10 fois l’entrée in-alpha à l’entité Alpha.

  2. Délègue 10 fois l’entrée in-beta à l’entité Beta.

Condition de réussite

  1. L’entité Alpha reçoit 10 fois la sortie out-alpha dans un délai maximal d’une minute.

  2. L’entité Beta reçoit 10 fois la sortie out-beta dans un délai maximal d’une minute.
